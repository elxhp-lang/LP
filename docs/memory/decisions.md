# 项目关键决策日志（ADR 轻量版）

## 记录规范
- 每条决策必须包含：背景、决策内容、影响范围、生效时间、是否可回滚。
- 决策目标：可复用、可共享、可追溯。

## DEC-001：采用“轻量核心 + 插件生态”架构
- 日期：2026-04-24
- 背景：需要在多个行业快速扩展，避免核心代码膨胀。
- 决策：核心仅保留认证、RBAC、插件生命周期管理、AI 调度；业务全部插件化。
- 影响范围：`core/`、`plugins/`、平台治理与发布流程。
- 生效状态：已生效
- 回滚策略：不建议回滚；仅允许在不破坏插件边界前提下局部优化。

## DEC-002：双端与后端技术栈固定
- 日期：2026-04-24
- 背景：需兼顾交付效率、工程成熟度与生态可用性。
- 决策：
  - Web：Next.js 15 + React + TypeScript（App Router）
  - Mobile：React Native + TypeScript（备选 PWA）
  - Backend：FastAPI + PostgreSQL + Redis
- 影响范围：全仓库技术实现与团队招聘协作。
- 生效状态：已生效
- 回滚策略：如需变更，必须先评审并更新记忆库主文档。

## DEC-003：插件通信与安全边界
- 日期：2026-04-24
- 背景：需要标准化插件通信并控制权限风险。
- 决策：通信遵循 MCP；插件运行在 WebAssembly 沙箱；仅白名单 API 可访问。
- 影响范围：插件 SDK、运行时、审核流程、安全策略。
- 生效状态：已生效
- 回滚策略：不可直接回滚，只允许增强隔离与审计能力。

## DEC-004：AI 调度统一下沉到服务端
- 日期：2026-04-24
- 背景：端侧算力有限，且需要统一成本与密钥治理。
- 决策：端侧不做重型计算；多模型调用统一经 FastAPI 服务层，包含密钥托管、用量统计、额度控制。
- 影响范围：AI 网关、计费统计、客户端调用方式。
- 生效状态：已生效
- 回滚策略：不可回滚，仅可优化调度策略。

## DEC-007：协作规则主动沉淀与先确认机制
- 日期：2026-04-24
- 背景：用户希望长期协作更顺畅，并明确大模型无法完整记忆全部历史上下文。
- 决策：
  - 对用户提示词与需求，默认先做“隐患检查 + 更优方案评估”。
  - 若涉及关键行为或关键架构调整，先与用户确认再执行。
  - 新增协作偏好必须主动写入记忆库文档体系，作为后续对话与执行基准。
  - 沟通统一采用中文和通俗表达，默认面向无代码背景说明。
- 影响范围：需求澄清、方案输出、代码实施、回归说明与协作体验。
- 生效状态：已生效
- 回滚策略：不可直接回滚；仅可在用户确认后调整细节表述。

## DEC-008：GitHub 入库采用“确认后推送”流程
- 日期：2026-04-24
- 背景：用户希望减少手工操作，同时避免误提交或误推送。
- 决策：
  - 每次阶段性工作完成后，我先给出变更清单与影响说明。
  - 仅在用户明确回复“确认推送”后，才执行测试、提交、推送。
  - 推送后必须返回执行结果（测试情况、提交摘要、推送状态）。
- 影响范围：版本管理流程、发布节奏、协作透明度与安全性。
- 生效状态：已生效
- 回滚策略：如需变更流程，先由用户确认并同步更新记忆库。

## DEC-009：阶段二采用“前端/插件TS + 后端FastAPI”实施方案
- 日期：2026-04-25
- 背景：阶段二需求同时提出“后端 FastAPI”与“统一 TypeScript”，存在实现冲突。
- 决策：
  - 保持既定技术栈不变：后端继续使用 FastAPI（Python）。
  - TypeScript 统一用于 Web、插件模板与共享类型层（`core/types`）。
  - 通过接口契约与模型对齐规则控制双语言协作一致性。
- 影响范围：阶段二代码骨架、后续接口演进、团队协作分工。
- 生效状态：已生效
- 回滚策略：如需改为全 TypeScript 后端，必须先完成架构评审与文档升级。

## DEC-010：阶段三插件生态以“生命周期闭环 + 权限隔离”作为MVP验收基线
- 日期：2026-04-26
- 背景：阶段三需要同时交付插件模板、示例插件与可演示运行链路，避免只停留在静态代码。
- 决策：
  - MVP 验收必须覆盖插件全生命周期：安装、配置、使用、卸载。
  - 权限策略采用“manifest 声明 + 运行时拦截”，未授权 API 调用必须被拒绝。
  - 提供本地沙箱脚本，作为开发调试与回归验证的统一入口。
- 影响范围：`core/plugin-sdk`、`plugins/examples`、`plugins/sandbox`、后端插件路由与加载器。
- 生效状态：已生效
- 回滚策略：可迭代增强，但不得取消生命周期闭环与权限隔离这两项底线。
